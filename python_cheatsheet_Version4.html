basic pattern
Pythonwhen to use
try: ... except SomeError:when code might fail and you want to recover
except ValueError:when converting input like int("abc")
except IndexError:when list access may go out of range
except KeyError:when a dict key may be missing
except TypeError:when an operation gets the wrong type
multiple exceptions
Pythonwhen to use
except (KeyError, TypeError):same recovery for related failures
except (IndexError, KeyError, ValueError):one fallback for several problems
except Exception as e:generic fallback when you want the error message
else — runs only if no exception
Pythonmeaning
try:
num = int(x)
except ValueError:
print("bad")
else:
print(num)		execute success-only logic
try:
result = a / b
except ZeroDivisionError:
msg = "error"
else:
msg = f"ok: {result}"		keep normal path separate from errors
finally — always runs
Pythonmeaning
try:
risky()
finally:
cleanup()		guarantee cleanup even after an error
try:
f = open("file")
...
finally:
f.close()		resource release
